# Documentation index

This directory is reserved for **maintained, reusable documentation** that matches the current codebase and contributor workflow.

If you are onboarding, start with:

1. [`CONTRIBUTOR_RUNBOOK.md`](CONTRIBUTOR_RUNBOOK.md) for the canonical setup + validate workflow.
2. [`USER_README.md`](USER_README.md) for product-facing usage notes.

## Generated or artifact-oriented subdirectories

- `docs/assets/qa-screenshots/` is a placeholder for curated QA screenshots.
- `docs/assets/siteplan/` is generated by `scripts/site_snapshot.py`.

Avoid storing one-off planning docs, issue-only execution checklists, and ephemeral payload artifacts in `docs/`; keep those in issue/PR context or regenerate as needed.
